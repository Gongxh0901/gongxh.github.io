# 个人网站首页内容规划方案

## 🎯 方案目标

一个顶级的游戏开发者个人主页，应该像一个引人入胜的故事，从"我是谁"到"我做过什么"再到"我的深度思考"，最后引导"如何找到我"。

**核心定位：** 资深游戏技术开发 + creator插件开发 + 技术分享

**目标用户：**
- 游戏公司技术负责人
- 游戏开发者同行

---

#### **1. 视觉引导区 (Hero Section) 🎮**

这是用户第一眼看到的内容，必须在3-5秒内抓住眼球，并清晰传达你的身份。作为游戏开发者，这里要体现技术专业性和视觉冲击力。

*   **🎯 核心要素:**
    *   **动态背景 (Dynamic Background):** 
        *   **首选:** 代码雨动画
    
    *   **响亮的头衔 (Headline) + 打字机效果:**
        *   主标题: `👋 你好，我是 gongxh`
        *   副标题: `🎮 游戏技术架构师 & 性能优化专家`
        *   描述: `深耕Cocos Creator引擎10年+ | 打造过5款获奖精品游戏 | 1000+开发者的工具选择`
    
    *   **精炼的个人简介 (Sub-headline):** 
        > "用代码雕琢游戏体验，用工具赋能开发团队。致力于将创意转化为现实，
        > 让复杂的技术变得简单易用。从比特小队到宫爆老奶奶，我的代码让百万玩家的快乐成为可能。"
    
    *   **关键行动按钮 (Call to Action - CTA):**
        *   **主按钮:** `🎯 查看我的游戏项目` (大按钮，突出显示)
        *   **次按钮:** `💬 技术交流` (微信/邮箱弹窗)
    
    *   **专业社交链接矩阵:**
        ```
        [GitHub] [Gitee] [微信] [邮箱] [技术博客]
        ```
        每个图标加 hover 动效，显示平台名称

*   **🎨 设计增强:**
    *   **个人头像:** 放置在左侧或右侧，圆形裁剪，带呼吸动效
    *   **技术标签云:** 环绕头像显示核心技术栈
    *   **成就数字:** 如 "10+ 年经验" "5+ 获奖项目" "1000+ Star"

    **🏷️ 项目信息区:**
    *   **项目标题 + 获奖徽章:**

## 🎨 设计与交互建议

### **🌙 整体视觉风格**

*   **主题色调:** 
    - **暗色主题 (推荐):** 深蓝/深灰背景 + 蓝绿色彩点缀，体现技术专业感
    - **亮色主题 (备选):** 白色背景 + 蓝色系，简洁现代
    - **主题切换:** 提供暗色/亮色一键切换功能

*   **配色方案:**
    ```
    主色调: #2196F3 (科技蓝)
    辅助色: #00BCD4 (青绿色) 
    强调色: #FF9800 (橙色，用于CTA按钮)
    文字色: #FFFFFF / #333333 (根据主题)
    背景色: #1A1A1A / #FAFAFA
    ```

*   **字体系统:**
    - **标题:** "PingFang SC", "Microsoft YaHei", sans-serif (中文)
    - **正文:** "SF Pro Display", "Helvetica Neue", sans-serif (英文)
    - **代码:** "Fira Code", "Monaco", monospace (技能条/代码展示)

### **⚡ 动效与交互**

*   **页面滚动动效:**
    - **Intersection Observer:** 元素进入视窗时触发动画
    - **淡入上浮:** 各区块从下方50px淡入，持续0.6s
    - **交错动画:** 同组元素依次出现，间隔0.1s
    - **视差滚动:** Hero区背景轻微视差效果

*   **悬停交互效果:**
    - **项目卡片:** 悬停上浮8px + 阴影增强
    - **技能条:** 悬停显示具体数值 + 进度动画
    - **社交图标:** 悬停放大1.2倍 + 旋转效果
    - **按钮:** 悬停渐变背景变化 + 轻微脉冲

*   **加载与性能:**
    - **懒加载:** 图片/视频进入视窗才加载
    - **预加载:** 关键资源优先加载
    - **骨架屏:** 内容加载时显示占位结构

### **📱 响应式设计**

*   **断点设置:**
    ```
    手机端: < 768px  → 单列布局，简化交互
    平板端: 768-1024px → 双列布局，保留主要功能  
    桌面端: > 1024px → 完整布局，所有交互效果
    ```

*   **移动端优化:**
    - **触摸友好:** 按钮最小44px，间距充足
    - **手势支持:** 项目卡片支持左右滑动浏览
    - **性能优先:** 动效简化，视频替换为GIF
    - **快速加载:** 图片压缩，关键路径优化

### **🚀 性能与SEO**

*   **性能优化:**
    - **首屏时间:** < 3秒加载完成
    - **图片优化:** WebP格式 + 响应式尺寸
    - **代码分割:** 按需加载非关键CSS/JS
    - **CDN加速:** 静态资源使用CDN

*   **SEO友好:**
    - **语义化HTML:** 正确使用header、section、article标签
    - **Meta标签:** 完善的title、description、keywords
    - **结构化数据:** JSON-LD格式的个人/作品信息
    - **友好URL:** 语义化的锚点链接

### **🔧 技术实现建议**

*   **框架选择:**
    - **Jekyll + SCSS:** 保持现有技术栈，静态生成
    - **JavaScript:** 原生JS实现交互，避免框架复杂性
    - **CSS Grid/Flexbox:** 现代布局方案

*   **第三方集成:**
    - **统计分析:** Google Analytics / 百度统计
    - **评论系统:** Gitalk / Valine (如果需要)
    - **邮件服务:** EmailJS (联系表单)

### **🎯 用户体验细节**

*   **导航体验:**
    - **锚点导航:** 平滑滚动到对应区域
    - **进度指示:** 显示页面阅读进度
